# OpenAI Data Export Structure

When you export your data from ChatGPT (Settings → Data Controls → Export Data), you receive a zip file containing:

| Item | Description |
|------|-------------|
| `conversations-*.json` | All ChatGPT conversations in structured JSON — messages, roles, timestamps, model used |
| `chat.html` | Full conversation history rendered as a single browsable HTML page |
| `export_manifest.json` | Index linking all exported items and their file references |
| `user.json` | Your OpenAI account profile data |
| `user_settings.json` | ChatGPT preferences and configuration |
| `shared_conversations.json` | Conversations you shared via public links |
| Top-level images | Files you uploaded to or received from ChatGPT during conversations |
| UUID folders | Per-conversation attachment bundles — screenshots, photos, audio messages |
| `dalle-generations/` | All images generated by DALL-E |
| `.md` / `.pdf` files | Documents you uploaded during conversations |

## Conversation JSON Structure

Each conversation in `conversations-*.json` is an object with:

```json
{
  "id": "uuid",
  "title": "Conversation title",
  "create_time": 1234567890.0,
  "update_time": 1234567890.0,
  "mapping": {
    "node-uuid": {
      "message": {
        "author": { "role": "user|assistant|system|tool" },
        "content": { "parts": ["message text"] },
        "create_time": 1234567890.0
      }
    }
  }
}
```

Key fields:
- `mapping` contains all messages as a tree (not a flat list) — each node has a parent
- `parts` is an array; text messages are strings, file references are objects
- `author.role` distinguishes user input, AI responses, system prompts, and tool calls
- System messages with >100 chars typically indicate a Custom GPT was used

## What Matters for Knowledge Distillation

The only files you need for the pipeline are the `conversations-*.json` files. Everything else (images, audio, HTML) is attachments and can be archived separately.
